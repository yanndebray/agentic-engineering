MATLAB Plain Text Live Script Creation

Overview

This skill enables transformation of markdown content (blog posts, documentation, tutorials) into MATLAB plain text Live Script format (.m). The plain text Live Code file format was introduced in MATLAB R2025a and combines the benefits of both worlds: rich Live Editor features with source control friendly plain text.

Reference Resources

For additional prompts and best practices for AI-assisted MATLAB development, see: - MATLAB Prompts Repository: https://github.com/matlab/prompts - MATLAB Skills Repository: https://github.com/matlab/skills

When to Use This Skill

Use this skill when: - Converting a markdown blog post to a MATLAB Live Script - Creating educational MATLAB content with formatted text and code - Generating interactive notebook-style files - Producing MATLAB documentation that works in the Live Editor

Plain Text Live Script Format (.m)

Key Characteristics

  • File Extension: .m (same as regular MATLAB scripts)
  • Opens In: Live Editor by default (can be opened as plain text)
  • Features Supported: All Live Editor features including output, formatted text, equations, images, controls, and tasks
  • Source Control: Fully compatible with Git, GitHub, GitLab, etc.

Structure Overview

Plain text Live Scripts use custom markup saved as MATLAB comments: 1. Inline markup: %[text], %[output], %[control], etc. 2. Appendix: Stored at the end of file for larger data (outputs, images, metadata)

Markup Syntax Reference

Formatted Text

%[text] This is basic text
%[text] # This is a title
%[text] ## This is a heading
%[text] ### This is a subheading
%[text] This is **bold**, *italic*, `monospace`, and <u>underlined</u>
%[text]{"align":"center"} This text is center aligned

Section Breaks

Section breaks are created using %% on its own line, with the title in a separate %[text] line:

%%
%[text] ## Section Title
%[text]
%[text] Section description text here.

code_here = 1;

%%
%[text] ## Next Section

IMPORTANT: Do NOT put the title on the same line as %%

%% Section Title     % WRONG - exports as code block in markdown

%%                   % CORRECT - clean section break
%[text] ## Section Title

When %% Title is used, the title appears as a code block in markdown export:

\`\`\`matlab
%% Section Title
\`\`\`

Heading Levels in Markdown Export

Heading levels in %[text] are promoted by one level when exported to markdown: - %[text] ## Heading → exports as # Heading - %[text] ### Subheading → exports as ## Subheading

Use one level deeper than your intended markdown output:

%%
%[text] ## Main Section      % Exports as # Main Section
%[text]
%[text] ### Subsection       % Exports as ## Subsection
%[text]
%[text] #### Details         % Exports as ### Details

MATLAB code example with syntax highlighting:

Best Practices

DO:

  • Use %[text] ## Heading for section titles (NOT %% Section Title)
  • Use %% alone (without title) only when you need section breaks for Live Editor navigation
  • Keep code executable - test all code before finalizing
  • Use meaningful output IDs (e.g., plot001, result001)
  • Add blank %[text] lines for visual spacing
  • Escape special characters in text markup when needed

DON'T:

  • Don't use %% Section Title - this exports as a code block in markdown
  • Don't use %[text] prefix for actual MATLAB code
  • Don't forget the %[appendix] and %[metadata:view] at the end
  • Don't use .mlx extension - this skill creates .m plain text files
  • Don't include raw markdown syntax without converting it
  • Don't use unsupported LaTeX commands like \xrightarrow - use simpler alternatives like \longrightarrow

Handling Special Cases

Multiple Figures

Each figure needs its own output marker with unique ID:

figure(1);
plot(x, y1);
%[output:fig001]

figure(2);
plot(x, y2);
%[output:fig002]

Inline Code References

%[text] Use the `plot` function to visualize data.

Mathematical Content

Convert markdown math to LaTeX: - Markdown $x^2$ → $x^2${"altText": "x squared"} - Markdown $$\int_0^1 x dx$$ → $$\int_0^1 x dx$${"altText": "integral"}

Code That Shouldn't Execute

For example code that's just for display:

%[text] ```matlabCodeExample
%[text] % This code is just an example
%[text] x = magic(3);
%[text] ```

File Naming Convention

  • Use descriptive names: signalAnalysisTutorial.m
  • Avoid spaces and special characters
  • Use camelCase or underscores: signal_analysis_tutorial.m

Best Practices

DO:

  • Use %[text] ## Heading for section titles (NOT %% Section Title)
  • Use %% alone (without title) only when you need section breaks for Live Editor navigation
  • Keep code executable - test all code before finalizing
  • Use meaningful output IDs (e.g., plot001, result001)
  • Add blank %[text] lines for visual spacing
  • Escape special characters in text markup when needed

DON'T:

  • Don't use %% Section Title - this exports as a code block in markdown
  • Don't use %[text] prefix for actual MATLAB code
  • Don't forget the %[appendix] and %[metadata:view] at the end
  • Don't use .mlx extension - this skill creates .m plain text files
  • Don't include raw markdown syntax without converting it
  • Don't use unsupported LaTeX commands like \xrightarrow - use simpler alternatives like \longrightarrow

  • Use camelCase or underscores: signal_analysis_tutorial.m
